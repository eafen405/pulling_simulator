02

模块接口 & 调用流向

main
状态模块
算法模块
卡池模块
记录模块
UI 模块
算法模块 → 调用 → 卡池模块（获取概率/角色列表） ← 唯一例外：只读查询
单向调用 总是 main 调用各模块，模块之间不互相调用
（算法调卡池是唯一例外，且是只读查询）
数据流向 状态数据：文件 ⇄ 状态模块 ⇄ main ⇄ 算法
静态数据：文件 → 卡池模块 → 算法
记录数据：main → 记录模块 → 文件
算法纯函数 不碰文件，不存状态
同样的输入永远得到同样的输出
